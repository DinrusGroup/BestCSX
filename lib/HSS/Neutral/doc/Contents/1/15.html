Remarks

A connection string requires at least the folder value. The other fields can be ommited.

Note for Silverlight 5: iso=[true|false] is defaulted to True if ommited. The iso value also indicates if your application is running in the browser or out of the browser. So if

C#
iso == true
would indicate your application is NOT running out of browser.

Sample connection strings for the different platforms

Windows Phone 7.5/8.0
  "name=myDbName;folder=myDbFolder;"
  "name=myDbName;folder=myDbFolder;password=askdashjkjsd;"
Windows RT
  "name=myDbName;folder=myDbFolder;"
  "name=myDbName;folder=myDbFolder;password=askdashjkjsd;"
Silverlight 5
  "name=myDbName;folder=myDbFolder" (In the Browser)
  "name=myDbName;folder=myDbFolder;password=askdashjkjsd" (In the Browser)
  "name=myDbName;folder=c:\\myDbFolder;iso=false" (Out of Browser)
  "name=myDbName;folder=c:\\myDbFolder;password=askdashjkjsd;iso=false;" (Out of Browser)
Mono for Android
  "name=myDbName;folder=" + Path.Combine(Environment.GetFolderPath(Environment.SpecialFolder.Personal), "myDbFolder")
  "name=myDbName;folder=" + Path.Combine(Environment.GetFolderPath(Environment.SpecialFolder.Personal), "myDbFolder") + ";password=askdashjkjsd;"
MonoTouch
  "name=myDbName;folder=" + Path.Combine(Environment.GetFolderPath(Environment.SpecialFolder.Personal), "myDbFolder")
  "name=myDbName;folder=" + Path.Combine(Environment.GetFolderPath(Environment.SpecialFolder.Personal), "myDbFolder") + ";password=askdashjkjsd;"
Microsoft .NET
  "name=myDbName;folder=c:\\myDbFolder;"
  "name=myDbName;folder=c:\\myDbFolder;password=askdashjkjsd;"
