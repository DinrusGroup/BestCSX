Remarks

NOTE: This is a single operation method and cannot be run from an existing transaction.

Removes all objects from a table, but the table structure and its columns, constraints, indexes and so on remain. The counter used by an identity for new rows is NOT reset. If you want to reset the identity counter, use Truncate<T> instead. If you want to remove the table definition and its data, use the DropTable<T> method.

This is the similar to SQL 'Delete * FROM' without a WHERE clause.

Increments the DataVersion for each object removed.
