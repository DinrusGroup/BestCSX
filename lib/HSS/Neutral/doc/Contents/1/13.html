Remarks

Connection string and constructor overloads vary depending on the platform. See DatabaseConnectionString for platform examples. An instance of the Database object represents a 'connection' to a folder. The folder is for all intents and purposes your database. Given that the HSS Database is a 'Client' database, there is no session or remote connection to a server, it runs in-process.

If you try to create a new database instance with a connection/folder of an already opened database (local or external) you will receive an exception. There can only be a single open database instance per connection folder. In order to support connection/database instance sharing, you can leverage the DatabaseConnectionPool. You can also create your own database context class to manage open connections for a given folder.

For each Type of object stored in the database, there is a corresponding file, logically a Table. Having a file for each Table allows for easy backup/restore and transporting copies of a single table's data if so desired.

The HSS Database provides support for ACID Compliant Transactions. If you do not wrap your calls in an explicit transaction, each individual call with run inside of an implicit transaction.
