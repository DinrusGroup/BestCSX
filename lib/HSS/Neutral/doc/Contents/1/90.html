Remarks

The DataVersion value is similar to MS SQL Change Tracking VERSION, in that everytime an object is added, updated or deleted, the version is incremented but the change itself is not captured or retained.

This is useful when the Database is used as a synchronization client where you want to set the DataVersion equal to the Version from your remote master data source. When using as a synchronization client you could delete all the objects first, then re-insert all the objects from the external master data source.

You could also use the SetRange<T> method to first update or add any modified objects to the database and then call the RemoveRange<T, TKey> or RemoveRange<T> method to delete any objects that have been deleted from the master data source and finally setting the DataVersion.

NOTE: The version is not incremented if the schema is changed.
